## 다른 로컬에서 셋업 (중요)

```bash
git clone https://github.com/baessu/calendar-agent.git
cd calendar-agent
npm install

# Vercel 프로젝트 연결 (외부 공유 기능에 필요한 BLOB 토큰을 받기 위함)
npx vercel link        # baessus-projects / calendar-agent 선택
npx vercel env pull    # .env.local 생성 → BLOB_READ_WRITE_TOKEN 주입

npm run dev            # http://localhost:3000
```

**왜 `vercel link` + `env pull`이 필요한가:**
- `.env.local`, `.vercel/`는 **gitignore**라 repo에 없음 (시크릿 보호).
- 외부 공유(`/api/share`, `/s/[token]`)는 `BLOB_READ_WRITE_TOKEN`이 있어야 동작. 없으면 공유 API가 503을 반환하고, 캘린더 본체 기능은 정상 동작.
- 토큰 없이 캘린더만 개발할 거면 `env pull` 생략 가능.

**폰트:** 영문 `Plain`(유료·자체호스팅 woff2)은 repo에 **미포함**(`public/fonts/` 없음). 없으면 IBM Plex Sans KR / 시스템 폰트로 graceful fallback. Plain을 쓰려면 `public/fonts/Plain-Regular.woff2`를 직접 넣으면 됨(`app/globals.css`의 `@font-face` 참고).

## 참고 / 함정 (gotchas)

- **Turbopack 캐시:** export 추가 후 dev에서 "Export X doesn't exist" 나오면 `rm -rf .next` 후 재시작.
- **시드 중복:** seeding은 rw 트랜잭션 내 empty-check로 StrictMode 더블이펙트 방어 (`lib/db/seed.ts`). 데이터가 2개씩 보이면 이 로직 확인.
- **공유 토큰 보안:** `.env.local`(BLOB 토큰)은 절대 커밋 금지(gitignore됨). 공개 페이지는 SSR이 서버에서만 Blob을 조회하므로 뷰어 브라우저에 토큰이 노출되지 않음.
- **인쇄:** 막대 `barIn` 애니메이션 opacity:0 시작 때문에 인쇄 스냅샷에서 안 보이던 이슈 → `@media print .print-cal/.share .cal-bar { animation:none; opacity:1 }`로 해결됨. 인쇄 관련 수정 시 주의.
- **Ralph 펜스:** 특정 스토리를 Ralph가 안 고르게 하려면 `prd.json`에서 우선순위 높은(번호 큰) 스토리를 임시 `passes:true`로. 작업 후 반드시 되돌릴 것.
