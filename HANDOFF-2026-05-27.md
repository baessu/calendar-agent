## 다른 로컬에서 셋업 (중요)

```bash
git clone https://github.com/baessu/calendar-agent.git
cd calendar-agent
npm install

# Vercel 프로젝트 연결 (외부 공유 기능에 필요한 BLOB 토큰을 받기 위함)
npx vercel link        # baessus-projects / calendar-agent 선택
npx vercel env pull    # .env.local 생성 → BLOB_READ_WRITE_TOKEN 주입

npm run dev            # http://localhost:3000
```

**왜 `vercel link` + `env pull`이 필요한가:**
- `.env.local`, `.vercel/`는 **gitignore**라 repo에 없음 (시크릿 보호).
- 외부 공유(`/api/share`, `/s/[token]`)는 `BLOB_READ_WRITE_TOKEN`이 있어야 동작. 없으면 공유 API가 503을 반환하고, 캘린더 본체 기능은 정상 동작.
- 토큰 없이 캘린더만 개발할 거면 `env pull` 생략 가능.

**폰트:** 영문 `Plain`(유료·자체호스팅 woff2)은 repo에 **미포함**(`public/fonts/` 없음). 없으면 IBM Plex Sans KR / 시스템 폰트로 graceful fallback. Plain을 쓰려면 `public/fonts/Plain-Regular.woff2`를 직접 넣으면 됨(`app/globals.css`의 `@font-face` 참고).

  - US-022 전체 뷰 히트맵(모노크롬 농담)
  - US-023 공유 스냅샷 발행 (`@vercel/blob`, `/api/share`)
  - US-024 공개 읽기전용 페이지 `/s/[token]` (SSR, 인쇄, not-found)
  - US-025 공유 관리 UI (상단바 '공유' → 발행/복사/갱신/해제, Dexie v5 `shares` 테이블)
- **v3 (편집 링크 공유): 완료·배포.** 읽기전용에 더해 협업 편집을 추가.
  - **두 토큰 모델:** 공유마다 `viewToken`(읽기 `/s/{view}`) + `editToken`(쓰기 `/e/{edit}`). 소유자는 항상 editToken 보유(갱신·해제 인증), 협업자에게 편집 링크 전달 시 편집 허용.
  - **편집 화면 `/e/{editToken}`:** `CalendarView`를 스냅샷 상태 위에서 재사용(`EditableSharedCalendar`). 일정·마커 생성/수정/이동/삭제 가능(프로젝트·태스크종류 관리는 소유자 전용). 변경마다 디바운스(0.8s) 자동 저장 + 저장 상태 표시. 재공유 불가.
  - **소유자 '가져오기'(pull):** 협업자 저장 시 스냅샷 `publishedAt`이 바뀜 → 소유자 앱이 자기 기록과 비교해 공유 버튼 `•` 배지 + 팝오버 "협업자가 편집했어요 · [변경 가져오기]". 가져오면 Blob→로컬로 해당 프로젝트 일정·마커 통째 교체(마지막-저장-우선). '갱신'(push)은 로컬→Blob 덮어쓰기(경고 문구).
  - **보안:** 모든 쓰기/삭제에 editToken 필수. 보기 링크가 유출돼도 읽기만 가능(이전엔 viewToken만으로 덮어쓰기/해제 가능했던 허점 해소).
  - **실시간 아님:** 폴링 없이 앱/팝오버 열 때 1회 비교. `publishedAt`은 클라이언트 시계 기반.
- **추가 수정:** 태스크종류 톤 비활성화가 레거시 데이터에서도 동작(nearest-step 매칭). 편집 화면 스크롤 버그 수정(`.edit-shell .ed-main { min-height:0 }` — flex column에서 `.ed-calwrap`이 스크롤되도록).
- **품질:** typecheck/lint 클린, **테스트 253개 통과**, 프로덕션 빌드 OK.
- **검증:** 외부 공유 Blob 왕복(발행→공개페이지→해제)을 로컬 + **프로덕션 모두 통과**.
- **배포 상태:** ✅ 외부 공유 포함 최신(`d955e2e`)이 프로덕션(sigma)에 배포됨. `BLOB_READ_WRITE_TOKEN`은 Production env에 설정 완료.

**공유 Blob 레이아웃:** 스냅샷 `shares/{viewToken}.json` + edit-key 포인터 `shares/keys/{editToken}.json → {viewToken}`. editToken은 포인터로 viewToken을 해석해 쓰기 인증. 레거시 공유(editToken 없음)는 '갱신' 시 서버가 editToken을 자동 발급(adoption)해 업그레이드.

## 참고 / 함정 (gotchas)

- **Turbopack 캐시:** export 추가 후 dev에서 "Export X doesn't exist" 나오면 `rm -rf .next` 후 재시작. **CSS도 마찬가지** — `globals.css` 수정이 dev에 반영 안 되면(스타일 그대로면) `rm -rf .next` + dev 재시작. (편집 화면 스크롤 수정 때 이걸로 막혔었음.)
- **Vercel 배포 차단 — 커밋 작성자 이메일:** git이 자동 설정한 `user@host.local`(호스트네임 기반) 이메일로 커밋하면 Vercel이 "commit author email not valid"로 빌드를 **차단**함 (status UNKNOWN, 빌드 로그 비어 있음, 시작도 안 함 — 플랫폼 장애처럼 보임). 유효 이메일로 커밋해야 함. 이 레포는 `git config user.email "54324311+baessu@users.noreply.github.com"`로 **로컬 설정**됨(전역 미변경). 다른 머신/레포에서 재발 가능.
- **좀비 배포가 빌드 슬롯 점유:** 차단/멈춘 UNKNOWN 배포가 이후 프로덕션 빌드를 막을 수 있음. `vercel ls`로 확인 후 `vercel remove <url> --yes`로 정리(라이브 별칭이 가리키는 Ready 배포는 건드리지 말 것).
- **시드 중복:** seeding은 rw 트랜잭션 내 empty-check로 StrictMode 더블이펙트 방어 (`lib/db/seed.ts`). 데이터가 2개씩 보이면 이 로직 확인.
- **공유 토큰 보안:** `.env.local`(BLOB 토큰)은 절대 커밋 금지(gitignore됨). 공개 페이지는 SSR이 서버에서만 Blob을 조회하므로 뷰어 브라우저에 토큰이 노출되지 않음.
- **인쇄:** 막대 `barIn` 애니메이션 opacity:0 시작 때문에 인쇄 스냅샷에서 안 보이던 이슈 → `@media print .print-cal/.share .cal-bar { animation:none; opacity:1 }`로 해결됨. 인쇄 관련 수정 시 주의.
- **Ralph 펜스:** 특정 스토리를 Ralph가 안 고르게 하려면 `prd.json`에서 우선순위 높은(번호 큰) 스토리를 임시 `passes:true`로. 작업 후 반드시 되돌릴 것.
